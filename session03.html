Error messages

Error messages

  • As a: user who submits a form
  • I want: to know what errors I’ve made, where I have made those errors and how to fix them
  • So that: I can fix errors quickly and easily

3.3.1 Error Identification (A)

If an input error is automatically detected, the item that is in error is identified and the error is described to the user in text

Semi-automated test | Dev tools | Screen reader

3.3.3 Error Suggestion (A)

If an input error is automatically detected and suggestions for correction are known, then the suggestions are provided to the user, unless it would jeopardize the security or purpose of the content

Semi-automated test | Dev tools |Screen reader

1. Multiple errors within a form

If there are one or more errors within a form, on submit should trigger either:

  • focus should be set on the first error within the form OR
  • focus should go to an overall error message above the form

In the case of an overall error message above the form:

  • Set focus on the error message container
  • List all errors within error message container
  • Recommended to link each error to affected field

2. Inline errors

For each form field in a state of error:

  • Display an error message that helps users resolve the error
  • Programmatically associate the error message with the form control
  • Set the form control with aria-invalid="true"
<label for="n1">Name</label>
                                 <input
                                   type="text"
                                   id="n1"
                                   aria-describedby="e1"
                                   aria-invalid="true"
                                 >
                                 <span id="e1">Error message</span>
