Testing for accessibility

There are a wide range of automated testing tools that can be used to identify and address accessibility issues.

Automated desktop tools

  • Axe by Deque
  • Microsoft Accessibility Insights
  • ANDI tool
  • WAVE tool

Automated mobile tools

  • iOS Accessibility Inspector
  • Android Accessibility Scanner
  • Axe for Android
  • Android Studio - Lint

There are also many manual testing tools available that can be used to identify specific accessibility issues.

Assistance tools

  • Web Developer extension
  • HeadingMaps

Colour and contrast tools

  • Colour Contrast Analyser by TPGI
  • Contrast Ratio by Lea Verou
  • WebAIM Color Contrast Checker
  • Tanaguru contrast finder
  • Stark

Desktop screen readers

  • JAWS for Windows (licensed)
  • NVDA for Windows
  • VoiceOver for MacOS

Mobile screen readers

  • TalkBack (Android)
  • VoiceOver for iOS (Apple)

Desktop Speech Interfaces

  • Dragon NaturallySpeaking (licensed)
  • Voice Control (Mac OS)

Mobile Speech Interfaces

  • Voice Access (Android)
  • Voice Control (iOS)

While these tools can be beneficial for teams, they should not replace expert reviews, accessibility audits or inclusive user research and testing.
